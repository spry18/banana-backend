```text
# Field Selector Mobile App: UI-to-API Mapping

## 🔐 Global / App Initialization (Pre-Dashboard)
- **Login Screen**: `POST /api/users/login` (Sends `mobileNo` and `password`, returns the Bearer Token).
- **App Header/Profile**: `GET /api/users/me` (Fetches the user's name to display "Welcome back, [Name]" across the app).

## 🌅 Page 1: Morning Report
- **Action**: Submitting the start-of-day reading and advance.
- **API**: `POST /api/daily-logs/start`
- **Payload**: formdata containing `startKm`, `startKmPhoto`, and optionally `petrolAdvance` and `petrolReceiptPhoto`.

## 📊 Page 2: Dashboard (Home)
- **Action**: Displaying the KPI counters and Today's Activity feed.
- **API**: `GET /api/field-selector/dashboard` (Returns total, assigned, selected, rejected, missed, visited counts and recent history).

## 📋 Page 3: All Fields List
- **Action**: Viewing and searching the assigned farmer plots.
- **API**: `GET /api/field-selector/fields`
- **Notes**: Pass query parameters like `?status=Assigned` or `?page=1` to filter the tabs (All, Assigned, Selected, Rejected).

## 🚜 Page 4: Field Visit Data (The Inspection Form)
- **Action 1 (On Page Load)**: Fetch the dropdown options (Packing sizes, Harvesting times, etc.).
  - **API**: `GET /api/inspections/config`
- **Action 2 (On Page Load)**: Fetch the specific farmer and crop data to display at the top.
  - **API**: `GET /api/field-selector/fields/:id` (Using the ID of the clicked enquiry).
- **Action 3 (On Submit)**: Sending the composition percentages, 20 photos, and final decision.
  - **API**: `POST /api/inspections` (Payload mapped safely to the DB, sending exactly 'SELECTED' or 'REJECTED').

## 🌙 Page 5 & 6: EOD Report & Success Screen
- **Action 1 (Submit Report)**: Logging the final odometer reading to calculate total distance.
  - **API**: `PATCH /api/daily-logs/end` (Sends `endKm` and `endKmPhoto`).
- **Action 2 (Logout)**: Securely ending the session from either the form page or the success page.
  - **API**: `POST /api/users/logout`
```
